  ## Round 4 — Piped API Primary Source + Cobalt v9/v10 Multi-Tier Fallback (2026-05-02)
  Root cause: Previous cobalt.ts was sending Cobalt v10 request format (downloadMode:"audio") to ALL
  instances, but community instances run Cobalt v9 (different endpoint /api/json, different request
  body). This caused ALL instances to fail with 400 errors, including api.cobalt.tools which now
  requires JWT auth.

  ### artifacts/spotd-mobile/utils/cobalt.ts (complete rewrite):
  - Tier 1 — Piped API (5 instances: pipedapi.kavin.rocks, pipedapi.adminforge.de, pipedapi.tokhmi.xyz, api.piped.yt, pipedapi.reallyaweso.me): Proxied YouTube audio streams, no auth needed. Returns M4A (preferred for Android compatibility) or WEBM.
  - Tier 2 — Cobalt v9 (4 instances: cobalt.api.timelessnesses.me, cobalt.tools.exozy.me, cobalt.privacydev.net, co.wuk.sh): Uses legacy POST /api/json endpoint with {isAudioOnly:true, aFormat:"mp3", audioBitrate} body. No auth required on most instances.
  - Tier 3 — Cobalt v10 (3 instances: cobalt.canine.tools, cobalt.hectabit.org, api.cobalt.tools): Uses new POST / endpoint with {downloadMode:"audio", audioFormat:"mp3"} body. Last resort.
  - extractVideoId() helper added to parse videoId from YouTube URLs for Piped.
  - getCobaltVideoUrl() skips Piped (video-only streams) and uses Cobalt v9 → v10.

  ### artifacts/spotd-mobile/contexts/DownloadContext.tsx (3 extension fixes):
  - Line 141: Spotify track downloads — replaced hardcoded .mp3 extension with cobalt.filename.split(".").pop() so M4A files from Piped get the correct .m4a extension.
  - Line 286: YouTube single video/audio downloads — same fix.
  - Line 389: YouTube playlist downloads — same fix.
